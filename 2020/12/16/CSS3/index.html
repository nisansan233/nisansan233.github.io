    十.伪类和伪元素

    伪类(pseudo-classes)

  • 常见的伪类
  • 动态伪类(dynamic pseudo-classes)
    • :linkvisited、:hover、:active、:focus
      • link初始化的状态
      • visited被访问过的状态
      • hover鼠标悬浮/停留状态(必须放在:link和:visited之后)
      • active正在访问状态,按下去没松开(必须放在:hover之后)
      • :focus指当前拥有输入焦点的元素(能接收键盘输入)
        • 去除元素的默认:focus效果
          1. outline: none
          2. tabindex=”-1”
      • 建议顺序为:link,:visit,:focus,:hover,:active
  • 目标伪类
  • 语言伪类
  • 元素状态伪类
  • 结构伪类(structural pseudo-classes)
    • :nth-child(),:nth-last-child(),:nth-of-type(),nth-last-of-type()
      • :nth-child(n) 选择器匹配属于其父元素的第 N 个子元素，不论元素的类型，n 可以是数字、关键词或公式。
      • :nth-of-type(n) 选择器匹配属于父元素的特定类型的第 N 个子元素的每个元素，n 可以是数字、关键词或公式。
    • :first-child,:last-child,:first-of-type
    • :root,:only-child,:only-of-type,:empty
      • :root根元素,也就是html元素
      • :only-child父元素中唯一的子元素
      • :only-of-type父元素中唯一的这种类型的子元素
        注意
      • n 为自然数
      • <html>和<body>也是页面元素标签
  • 否定伪类
    • :not()
      • :not()的格式是:not(x)
      • x是一个基础选择器

    十二.CSS特性

  • CSS属性的继承

    • 一个元素如果没有设置某属性的值,就会跟随父元素的值
    • 并不是所有的属性值都可以继承(文本相关的属性,更多的去查文档MDN)
    • 当然,如果设置了自己的属性值,就会使用自己的
      注意：
      • CSS属性继承的是计算值，并不是当初编写属性时的指定值(字面值)

  • CSS属性的层叠

    • CSS允许多个相同的名字的CSS属性出现在同一个元素上
    • 层叠后的结果是：相同的CSS属性中,只有一个CSS属性会生效

  • 基本层叠:选择器相同时，后面的属性会层叠(覆盖)前面的属性

  • 选择器的权重(优先级)

    • !important:10000
    • 内联样式:1000
    • id选择器:100
    • 类选择器、属性选择器、伪类:10
    • 元素选择器、伪元素:1

CSS属性的使用经验

  • 有时编写的CSS属性无法生效
    • 选择器优先级太低
    • 选择器没选中对应的元素
    • CSS属性的使用形式不对
      • 元素不支持此CSS属性，比如span默认是不支持widtn和height的
      • 浏览器不支持此CSS属性，比如旧版的浏览器不支持
      • 被同类型的CSS属性覆盖，比如font覆盖font-size
    • 充分使用浏览器的开发工具进行调试

十三.CSS属性

  • CSS属性-display
    • block
      • 让元素显示为块级元素
    • inline
      • 让元素显示为行内级元素
      • 行内元素无法设置宽高
    • none
      • 隐藏元素
    • inline-block
      • 让元素同时具备行内级元素,块级元素的特征
      • 跟其他行内级元素在同一行显示
      • 可以随意设置宽高
      • 宽高默认由内容决定
      • 对内是行内级元素，对外是块级元素
      • 用来让行内非替换元素能够随时设置宽高
      • 让块级元素能够跟其他元素在一行显示
    • dispaly的以下取值，等同于某些HTML元素
      • table:<table>，一个block-level表格
      • inline-table:<table>，一个inline-level表格
      • table-row:<tr>
      • table-row-group:<tbody>
      • table-header-group:<thead>
      • table-footer-group:<tfood>
      • table-cell:<td>,<th>,一个单元格
      • table-caption:<caption>,表格的标题
      • list-item:<li>
    • CSS属性-visibility
      • visibility,能控制元素的可见性,有两个常用值
      • visible:显示元素
      • hidden:隐藏元素
      • visibility:hidden;和display:none的区别
        • visiblity:hidden；
          • 虽然看不见，但元素的框架依旧还留着,还会占着原来的位置
        • display:none
          • 不仅元素看不见,而且元素的框也会被移除,不会占着任何位置
    • CSS属性-overflow
      • overflow用于控制内容溢出行为
        • visible:溢出的内容依然可见(默认)
        • hidden:溢出部分隐藏起来
        • scroll:溢出的内容被裁剪,但可以通过滚动机制查看
          • 会一直显示滚动条区域,滚动条区域占用的空间属于width,height
        • auto:自动根据内容是否溢出来决定是否提供滚动机制
    • 强调两个知识点
      • 元素中的空格
        • 行内级元素(包括inline-block元素)的代码之间如果有空格,会被解析显示为空格
        • 目前建议的解决方法
          1. 元素代码之间不要留空格
          2. 注释掉空格
          3. 设置父元素的font-size为0,然后在元素中重新设置自己需要的font-size
          4. 给元素加float
      • 块级元素,inline-block元素
        • 一般情况下,可以包含其他任何元素(比如块级元素,行内级元素,inline-block元素)
        • 但是行内元素里面不要嵌套块级元素
        • 行内元素一般只包含行内元素
    • 还有overflow-x,overflow-y两个属性,可以分别设置水平垂直方向
    • word-break属性(一般用不上)
      • 换行
    • CSS属性- border-***-radius
      • 设置边框为圆角
      • 第一个值是水平半径
      • 第二个值是垂直半径
      • 还可以直接设置百分比
        • border-radius大于或等于50%,就会变成圆

          十四.盒子模型

    • HTML中的每个元素都可以看做是一个盒子
      • 内容(content)
        • 盒子里面装的东西
      • 内边距(padding)
        • 盒子边缘和里面装的东西之间的间距
      • 边框(border)
        • 就是盒子的边框,边缘部分
      • 外边距(margin)
        • 盒子和其他盒子的间距
    • 默认的盒子模型:

内容相关属性

- width:宽度
  - min-width:最小宽度,无论内容多少，宽度都大于或等于min-width
  - max-width:最大宽度,无论内容多少，宽度都小于或等于max-width
- height:
  - min-height:最小高度,无论内容多少，高度都大于或等于min-height
  - max-height:最大高度,无论内容多少，高度都小于或等于max-height

内边距相关属性

- padding-left:左内边距
- padding-right:右内边距
- padding-top:上内边距
- padding-bottom:下内边距
- padding:是padding-top,padding-right,padding-bottom,padding-left的简写属性
  - 按照顺时针方向取值top,right,bottom,left
  - 如果缺少bottom,那么bottom就会使用top的值
  - 如果缺少left,那么left就会使用right的值

外边距相关属性

- margin-left:左外边距
- margin-right:右外边距
- margin-top:上外边距
- margin-bottom:下外边距
- margin:是margin-top,margin-right,margin-bottom,margin-left的简写属性
- 上下margin的传递
  - margin-top传递
    - 如果块级元素的顶部线和父元素的顶部线重叠,那么这个块级元素的margin-top值会传递给父元素
  - margin-bottom传递
    - 如果块级元素的底部线和父元素的底部线重叠,并且父元素的高度是auto,那么这个块级元素的margin-bottom值会传递给父元素
  - 如何防止传递
    1. 给父元素设置padding-top/padding-bottom 
    2. 给父元素设置border
    3. 触发BFC:
      - 浮动可以触发
      - 设置overflow为非visible   
 - 上下margin的折叠
   - 垂直方向上相邻的2个margin(top和bottom)有可能会合并为1个margin,这种现象叫做collapse(折叠)
   - 水平方向上的margin永远不会collpase
   - 折叠最终值为两个值进行比较,取较大值
  • 建议
    • margin一般用来设置兄弟元素之间的间距
    • padding一般用来设置父子元素之间的间距

        CSS属性-outline

    • outline表示元素外轮廓
      • 不占据空间
      • ，默认显示在border的外面
    • outline相关属性
      • outline-width
      • outline-style
      • outline-color
      • outline:outline-width,outline-style,outline-color的简写属性
    • 应用实例

        box-shadow

    • box-shadow属性可以设置一个或者多个阴影
    • 每个阴影用<shadow>表示
      • <shadow>=inset?&&<length>{2,4}&&<color>?
        • ?表示0个或者1个
        • <>表示要写属性值
        • {2，4}表示2个到4个数字
        • 第一个<length>:水平方向的偏移,正常往右偏移
        • 第二个<length>:垂直方向的偏移,正常往下偏移
        • 第三个<length>:模糊半径(blur radius)
        • 第四个<length>:延伸距离
        • <color>:阴影的颜色,如果没有设置,就跟随color属性的颜色
        • inset:外框阴影变成内框阴影

          text-shadow

    • <length>{2,4}&&<color>?
    • text-shadow用法类似box-shadow,用于给文字设置阴影
    • text-shadow同样适用于::first-line,::first-letter

      box-sizing

    • box-sizing用于设置盒子模型中宽高的行为
    • content-box
      • 含义是设置宽度和高度时只指定内容的宽高
      • padding,border都是布置在width,height外边
    • border-box
      • 含义是设置宽度和高度时是内容+内边距+边框宽度
      • padding,border都是布置在width,height里边

        水平居中的总结

    • 元素居中
      • 普通文本,行内元素,图片,行内块级元素居中都是设置父元素的text-align:center
      • 块级元素是设置margin: 0 auto;left:0,right:0(或者左右相等,left和right默认是auto)
  • margin的原理
    • 水平方向上的宽度是auto,所以子元素的margin-left,margin-right设置为auto可以平分宽度

      十五.背景设置

      CSS属性-背景

    • background-image
      • background-image用于设置元素的背景图片
      • 会盖在(不是覆盖)background-color的上面
      • 如果设置了多张图片
        • 设置的第一张图片将显示在最上面,其他图片按顺序层叠在下面
        • 注意:如果设置了背景图片后,元素没有具体宽高,背景图片是不会显示出来的
  • background-repeat
    • background-repeat用于设置背景图片是否要平铺
    • background-repeat:repeat(默认)
    • background-repeat:repeat-x 水平方向平铺
    • background-repeat:repeat-y 垂直方向平铺
    • background-repeat:no-repeat 不平铺
  • background-size
    • background-size用于设置背景图片的大小
      • auto:以背景图片本身大小显示
      • cover:对背景图片进行等比拉伸,让背景图片覆盖整个元素
      • contain:对背景图片进行等比拉伸,拉伸到一个方向的宽度(高度),不再进行拉伸,保持图片的宽高比
      • 具体大小,比如100px,或者相对于原背景的百分比
        • 一个值: 这个值指定图片的宽度，图片的高度隐式的为auto
        • 两个值,第一个值指定图片的宽度，第二个值指定图片的高度
  • background-position
    • background-position用于设置背景图片在水平,垂直方向上的初始位置
    • 水平方向可以设置:left,center,right
    • 垂直方向可以设置:top,center,bottom
    • 如果只设置了一个方向,另一个方向默认值是center
    • <length> 或 <percentage>。指定相对于左边缘的 x 坐标，y 坐标被设置成 50%
      • 注意background-position:100px 100px不是移动容器,定位到图片右100px,下100px的地方,而是将图片相对容器进行位移,移动的是图片,所以要是想显示右下角100px,100px位置的图片,应该background-position:-100px -100px
  • background-attachment
    • scroll:背景图片跟随元素一起滚动(默认值)
    • local:背景图片跟随元素以及元素内容一起滚动
    • fixed:背景图片相对于浏览器窗口固定
  • background
    • background是一系列相关属性的简写
      • 常用格式image position/size repeat attachment color
        • background-size可以省略,如果不省略,/background-size必须紧跟在position后面
        • 其他属性也可以省略,而且顺序随意
  • background-image和img的选择
    • 总结
      • img,作为网页的重要组成部分,比如广告图片,LOGO图片,文章配图,产品图片
      • background-image,可有可无

标准流(Normal Flow)

  • 标准流的特点
    • 默认情况下,元素都是按照normal flow进行排布
    • 块级元素会独占一行,从上向下按顺序排列
    • 行内元素会按照顺序排列,碰到父元素的边缘则会自动换行
    • 默认情况下,互相之间不存在层叠现象
  • 脱标元素的特点
    - 脱离标准流的元素:postion:fixed/absolute,float
  - 可以设置宽度高
  - 宽高默认由内容决定
  - 不再受标准流的约束
  - 不再给父元素汇报宽高
- 和display,float的关系
  - MDN文档
  - 脱标后都是块级block元素，宽高为自动,所以参考内容的宽高

注意:绝对定位,浮动都会让元素脱离标准流,以达到灵活布局

  • margin，padding定位
    • 在标准流中,可以使用margin，padding对元素进行定位
    • 其中margin还可以设置负数
    • 比较明显的缺点是:
      • 设置一个元素的margin或者padding,通常会影响到标准流中其他元素的定位效果
      • 不便于实现元素层叠的效果

        CSS属性-position

    • 利用position可以对元素进行定位,常用值有4个
      • static:静态定位
        • position属性的默认值
        • 元素按照normal flow布局
        • left,right,top,bottom没有任何作用
      • relative:相对定位
        • 元素按照normal flow布局
        • 可以通过left,right,top,bottom进行定位(这四个的值如果写百分比是相对于父元素)
        • 定位参照对象是自己原来的位置
      • abosolute:绝对定位
        • 元素会脱离normal flow
        • 可以通过left,right,top,bottom进行定位
        • 定位参照对象是最邻近的定位祖先元素
        • 没有设置宽度时,以内容为宽度
        • 定位元素(positioned element)
          • position值不为static的元素
          • 也就是position值为relative,absolute,fixed的元素
        • 子绝父相
          • 在绝大多数情况下,子元素的绝对定位都是相对于父元素进行定位
          • 如果希望子元素相对于父元素定位,又不希望父元素脱标,常用的解决方法是:
            • 父元素设置position:relative(让父元素成为定位元素,而且父元素不会脱离标准流)
            • 子元素设置position:absolute
            • 简称:”子绝父相”
      • fixed:固定定位
        • 元素会脱离normal flow
        • 可以通过left,right,top,bottom进行定位
        • 定位参照对象是视口(viewport)
        • 当画布滚动时,固定不动
    • 绝对定位技巧
      • 绝对定位元素(absolute positioned element)
        • position值为absolute或者fixed的元素
      • 对于绝对定位元素来说
        • 定位参照宽度=left+right+margin-left+margin-right+绝对定位元素的实际占用宽度
        • 定位参照高度=left+right+margin-left+margin-right+绝对定位元素的实际占用高度
      • 如果希望绝对定位元素的宽高和定位参照对象一样,可以给绝对定位元素设置以下属性
        • left:0,right:0,top=0,bottom:0,margin:0
          水平居中公式让盒子定位到父元素水平中心的位置(left:50%),再让盒子向左移动自身宽度的一半,垂直居中同理
    • 如果希望绝对定位元素的内容居中,给绝对定位元素设置以下属性
      • left:0,right:0,top:0,bottom:0,margin:auto
      • 或者 left:50%;top:50%;transform:translate(-50%,-50%)
  • 元素的层叠
    • CSS属性-z-index
      • z-index属性用于设置定位元素的层叠顺序(仅对定位元素有效)
        • 取值可以是正整数，负整数，0
        • 比较原则：
          • 如果是兄弟关系:
            • z-index越大,层叠越在上面
            • z-index相等,写在后面的元素层叠在上面
          • 如果不是兄弟关系
            • 各自从元素自己以及祖先元素中,找出最领近的两个定位元素进行比较
            • 而且这两个定位元素必须始终z-index的具体数值

十七.浮动float

  • 为什么需要浮动,浮动可以解决什么问题
    1. 如何让多个块级盒子(div)水平排列成一行？
      • 比较难,虽然转换为行内块级元素可以实现一行显示,但是他们之间会有很大的空白间隙,很难控制
    2. 如何实现两个盒子的左右对齐
    • float有三个常用的取值
      • none:不浮动
      • left:向左浮动
      • right:向右浮动
    • 浮动的特性
      1. 浮动元素会脱离标准流
      2. 浮动元素会一行内显示并且元素顶部对齐
      3. 浮动元素会具有行内块级元素的特性
    • 浮动的规则一:
      • 元素一旦浮动后
        • 脱离标准流
        • 朝着向左或者向右方向移动,直到自己的边界紧贴着包含块(一般是父元素)的边缘或者其他浮动元素的边界为止
      • 定位元素会层叠在浮动元素上面
    • 浮动的规则二：
      • 浮动元素不能和行内级内容层叠,行内级内容将会被浮动元素推出
      • 比如行内级元素,inline-block
    • 浮动的规则三:
      • 行内级元素,inline-block元素浮动后,其顶部将与所在行的顶部对齐
    • 浮动规则四:
      • 如果元素是向左(右)浮动,浮动元素的左(右)边界不能超出包含块的左(右)边界
    • 浮动规则五:
      • 浮动元素之间不能层叠
        • 如果一个元素浮动,另一个元素已经在那个位置了,后浮动的元素会紧贴着前一个浮动的元素
        • 如果水平方向剩余的空间不够显示浮动元素,浮动元素将向下移动,直到有充足空间为止
    • 浮动规则六:
      • 浮动元素的顶部不能超过包含块的顶端,也不能超过之前所有浮动元素的顶端
  • 浮动元素经常和标准流父级元素搭配使用
    • 为了约束浮动元素的位置,我们网页布局一般采取的策略是:
      • 先用标准流的父元素排列上下位置,之后内部子元素采取浮动排列左右位置,符合网页布局第一准则

  • 使用时遇见的问题

    • 在进行布局时,同一行多个元素摆放,设置margin-right让它们产生间距,那么最后一个margin-right的设置是多余的,会将最后一个元素挤到下一行
      • 解决办法一:每一行的最后一个元素,加一个class,通过class去除这个margin-right
      • 解决办法二:使用伪类选择器
      • 解决办法三:
      • .contaniner 固定的宽度>.wrap+负margin>各个item
        • width of containing block=content widht + margin-left+ margin-right+padding-left+padding-right+border 解决边框重合问题,给元素设置负margin

  • 清除浮动

    • 由于父级盒子很多情况下,不方便给高度,但是盒子浮动又不占有位置,最后父盒子高度为0时,就会影响下面的标准流盒子
    • 由于浮动元素不再占用原文档流的位置,使用它会对后面的元素排版产生影响

  • 清除浮动的本质

    • 清除浮动元素造成的影响
    • 如果父盒子本身有高度,则不需要清除浮动
    • 清除浮动之后,父级就会根据浮动的子盒子自动检查高度,父级有了高度,就不会影响下面的标准流了
      语法：
      选择器:{clear:属性值;}
      属性值
      • left 清除左侧浮动影响
      • right 清除右侧浮动影响
      • both 同时清除左右两侧浮动影响
        我们实际工作中,几乎只用clear:both;
        清除浮动的方法
    1. 额外标签也称隔墙法,是W3C推荐的做法
      1. 在浮动元素末尾添加一个空的块级元素标签,使用clear(新的空标签必须是块级元素)
    2. 父级添加overflow属性
      1. 可以给父元素添加overflow属性,将其设置为hidden,auto,scroll,注意是给父元素添加代码
    3. 父级添加after伪元素
      1. 是额外标签法的升级版,也是给父元素添加
        父元素::after{content:””;
        display:block;
        height:0;
        clear:both;
        visiblity:hidden;
        }
        父元素{
        *zoom:1;//兼容IE6,IE7
        }
    4. 父级添加双伪元素
      父元素::before,父元素::after
      {content:””;
       display:table;
      }
      父元素::after{
       clear:both;
      }
      父元素{
       *zoom:1;//兼容IE6,IE7
      }

      十八.CSS3-transform

  • transform:变形,转换

    • 2D 转换之移动 translate
      • 2D移动是2D转换里面的一种功能,可以改变元素在页面中的位置,类似定位。注：二维坐标x,y x正方向向右,y正方向向下
      • 语法：
        • transform:translate(x,y);
        • 也可以分开写transform:translateX(n);transform:translateY(n)
      • 重点:
        • 定义2D转换中的移动,沿着X和Y轴移动元素
        • 不会影响其他元素的位置
        • translate中的百分比单位是相对于自身元素的百分比
        • 对行内标签没有效果
      • 应用场景:让盒子实现水平和垂直居中
        • display:absolute;top:50%;left:50%;transform:translate(-50%,-50%)
    • 2D 转换之缩放scale
      • 缩放,顾名思义,可以放大和缩小.只要给元素添加上了这个属性就能控制它放大或者缩小
      • 语法:
        • transform:scale(x,y);
      • 注意
        • 注意其中宽度x和高度y用逗号隔开,写的是倍数,不跟单位
        • transform:scale(1,1):宽和高都放大到一倍,相对于没有放大
        • transform:scale(2,2):宽和高都放大到2倍
        • 只写一个参数，第二个参数则和第一个参数一样
        • transform:scale(0.5,0.5):缩小
        • sacle缩放最大的优势:可以设置旋转中心缩放,默认以中心点缩放,而且不影响其他盒子
    • 2D 转换之旋转 rotate
      • 2D旋转指的是让元素在2维平面内顺时针旋转或者逆时针旋转或逆时针旋转
      • 语法:
        • transform:rotate(度数)
      • 重点:
        • rotate里面跟度数,单位是deg 比如rotate(45deg)即为旋转45度
        • 角度为正时,顺时针,负时,为逆时针
        • 默认旋转的中心点是元素的中心点
    • 2D旋转中心点transform-origin
      • 我们可以设置元素的旋转中心点
      • 语法:
        • transform-origin: x y;
      • 重点:
        • 注意后面的参数x和y用空格隔开
        • x y默认转换的中心是元素的中心点(50% 50%)
        • 还可给x y设置像素或者方位名词
    • skew
    • transition
      • 过渡
        • transition:要过渡的属性 花费时间 运动曲线 何时开始;

  • 转换综合写法

    1. 同时使用多个转换,其格式为:transform:translate() rotate() scale()等
    2. 其顺序会影响转换效果。(先旋转会改变坐标轴方向)
    3. 当我们同时有位移和其他属性的时候,记得要将位移放到最前面

精灵图

  • CSS Sprite(精灵图)

    • 是一种CSS图象合成技术,将各种小图片合并到一张图片上,然后利用CSS的背景定位显示对应的图片部分
    • 作用：
      • 减少网页的http请求数量,请求加快网页响应速度,减轻服务器压力
      • 减小图片总大小
      • 解决了图片命名困扰,只需要针对一张图片

        字体图标

  • 轻量级:一个图标字体要比一系列的图像要小。一旦字体加载,图标就会马上渲染出来,减少服务器请求

  • 灵活性:本质其实是文字,可以很随意改变颜色,产生阴影,透明效果,旋转等

  • 兼容性:几乎支持所有的浏览器,请放心使用

vertical-align 属性的应用

  • CSS的vertical-align属性使用场景:经常用于设置图片或表单(行内块元素)和文字垂直对齐,只针对行内元素或者行内块级元素有效
    • 值
      1. baseline 默认,元素放置在父元素的基线上
      2. top 把元素的顶端和行内最高的元素顶端对齐
      3. middle 把此元素放置在父元素的中部
      4. bottom 把元素的顶部与行中最低的元素的顶端对齐
  • 解决图片底部默认空白间隙问题
    • 原因是行内块元素会和文字的基线对齐
      1. 给图片添加vertical-align:middle|top|bottom
      2. 把图片转化为块级元素 display:block

CSS3动画

  1. 先定义动画
  2. 再调用动画
  • 用keyframes定义动画(类似定义类选择器)
    @keyframes 动画名称{
    0%{
    width:100px;
    }
    100%{
    width:200px;
    }
    }
  • 元素使用动画
    div{
    animation-name:动画名称;
    animation-duration:持续时间;
    }
  • 动画序列
    • 0%是动画的开始,100%是动画的完成.这样的规则就是动画序列
    • 在@keyframes中规定某些CSS样式,就能创建由当前样式逐渐改变成新样式的动画效果
    • 动画是使元素从某一种样式逐渐变化为另一种样式的效果,可以改变任意多的样式任意多的次数
    • 清使用百分比来规定变化的时间,或使用关键词”from”和”to”,等同于0%和100%
